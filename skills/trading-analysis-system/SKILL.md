# Trading Analysis System

用这个 skill 生成纪律化的交易辅助分析。核心目标不是预测涨跌，而是把市场观察转成可验证、可执行、可复盘的情景计划。所有结论都必须带条件、失效线、仓位约束和复盘记录。

## 基本原则

- 区分 `观察`、`推演`、`执行计划`，不要把推演写成事实。
- 使用条件语言：例如“若守住...”，“若放量突破...”，“若跌破...则结构失效”。
- 不给无条件买卖指令。把方向判断转成触发条件、止损、目标、仓位和失效处理。
- 风控优先于目标：先写失效线、仓位上限、暂停条件，再写目标位。
- 波浪计数、趋势标签、概率估计都视为假设，不视为确定事实。
- 如果结构不清晰，列出候选结构或竞争路径，不要硬凑单一路径。
- 如果用户没有提供成交量、时间周期、盘口、资金费率、宏观或链上数据，不要编造证据。
- 默认怀疑回测、历史拟合和漂亮叙事。过去有效不代表未来有效，所有分析都必须留出失效和重算机制。

## 分析流程

1. **定义范围**
   - 标的、市场、当前价格，如果用户提供则写明。
   - 时间框架：日内、短线 `2-10天`、波段 `3-8周`，或用户指定周期。
   - 任务类型：仅观察、执行计划、风险复核、策略验证、交易复盘。

2. **建立市场结构**
   - 判断状态：推动、修正、震荡、突破、衰竭、假突破。
   - 拆分结构：高周期方向、中周期波段/回踩、低周期触发。
   - 如有数据，补充波动率、量能、流动性、资金费率或事件影响。

3. **标注关键价位**
   - 分成 `压力`、`支撑`、`失效`、`大失效`。
   - 每个价位都说明来源：前高/前低、斐波那契延伸/回撤、区间边界、成交密集区、均线、缺口、清算区或结构拐点。
   - 明确哪些价位决定情景切换。

4. **生成三路径场景**
   - 基准场景：当前证据下概率最高的路径。
   - 延伸/突破场景：趋势继续、突破扩展或加速。
   - 回撤/失败场景：结构走弱、跌破关键位或需要重算。
   - 每个场景必须包含：概率、触发条件、目标区、失效条件、下一步观察点。

5. **写执行计划**
   - 需要时同时给两套路由：
     - 突破型：触发、确认、分笔进场、止损、第一目标、第二目标。
     - 回踩型：支撑区、反转确认、分笔进场、止损、目标。
   - 必须写仓位上限。用户没给风险偏好时，默认保守。
   - 必须写止盈逻辑：分批兑现、移动止损、停止加仓条件。
   - 必须写不交易条件。

6. **记录预测**
   - 使用 `价格-时间-失效` 格式。
   - 写清楚什么情况证明本次分析错了。
   - 写清楚什么时候必须重算结构。
   - 保持可审计，方便后续对比预测和实际走势。

7. **质量检查**
   - 所有关键位是否都标注了属性？
   - 每个场景是否都有条件和失效线？
   - 是否有明确的不交易条件？
   - 仓位上限是否明确？
   - 概率是否合理且合计为 100%？
   - 是否避免了“必然、一定、稳了”等确定性语言？

8. **补充可审计字段**
   - 分析编号：`{{SYMBOL}}_{{DATE}}_A`。
   - 当前价格与时间戳。
   - 使用周期：例如 `1H / 4H / 日线 / 周线`。
   - 主路径概率、核心失效位、计划复盘时间。
   - 是否允许交易：`允许 / 观察 / 禁止`。
   - 最大计划仓位和触发前禁止动作。

## 输出结构

普通分析默认输出这些章节：

1. `结论先行`
2. `市场结构`
3. `关键价位汇总`
4. `三路径场景`
5. `执行计划`
6. `预测记录（价格-时间-失效）`
7. `盘前/盘中检查清单`
8. `策略质量评分`
9. `复盘计划`
10. `一句话执行建议`

当用户要求正式 Markdown 报告、模板化输出、或需要可长期保存的分析日志时，读取 `references/output-template.md`。

## 概率规则

- 不要给过度精确的概率，优先使用 `45% / 30% / 25%` 这种粗粒度比例。
- 证据越弱，概率越接近，不要在结构模糊时强行给高置信度。
- 概率不能替代失效条件。即使 60% 的场景，也必须有清晰失败线。

## 策略质量评分

每次正式报告都给出 25 分制评分。评分不是看涨看跌分，而是“是否值得执行”的质量分。

| 维度 | 分数 | 含义 |
|---|---:|---|
| 结构清晰度 | 0-5 | 趋势、波段、触发是否清楚 |
| 关键位可信度 | 0-5 | 支撑/压力/失效位是否有明确来源 |
| 风险收益比 | 0-5 | 目标、止损、仓位是否匹配 |
| 失效条件清晰度 | 0-5 | 错了以后是否知道该停、减、重算 |
| 执行可操作性 | 0-5 | 计划是否能被实际执行和复盘 |

默认解释：

- `21-25`：结构清楚，可执行但仍需按触发条件。
- `16-20`：可观察或小仓试错，禁止追高追空。
- `11-15`：信息不足或结构冲突，只能观察。
- `0-10`：禁止交易，优先等待新结构或新数据。

## 风控默认值

用户未提供风险规则时：

- 把短线计划视为战术计划，不视为全仓方向判断。
- 单一计划仓位上限默认不超过计划资金的 `20%-40%`。
- 分笔进场默认用 `60% + 40%` 或 `50% + 50%`，且后手必须等待确认。
- 快速突破后若重新跌回突破位下方，优先减仓、取消追入或重算。
- 大失效位跌破后，停止沿用当前结构，从高周期重新计算。

## 复盘模式

当用户提供历史分析或交易记录时：

- 对比实际走势与原来的三路径场景。
- 把结果归类为 `基准命中`、`延伸命中`、`失败命中`、`失效` 或 `模糊`。
- 判断错误来源：结构识别、关键位选择、时间窗口、仓位管理、执行纪律。
- 保留有价值的失败预测，因为失败样本是系统迭代的重要反馈。

复盘时使用这些字段：

| 字段 | 说明 |
|---|---|
| 场景命中 | 基准 / 延伸 / 失败 / 全部失效 / 模糊 |
| 关键位有效性 | 支撑/压力是否真的起作用 |
| 最大顺向波动 | 计划触发后最大浮盈空间 |
| 最大逆向波动 | 计划触发后最大浮亏空间 |
| 是否先扫损后到目标 | 检查止损是否太紧 |
| 是否到目标但没触发进场 | 检查入场条件是否太苛刻 |
| 错误类型 | 结构错 / 时间错 / 价位错 / 仓位错 / 执行错 |

## 三角色检查

复杂分析或重要交易计划必须模拟三个角色：

- `分析师`：输出结构、关键位、三路径和执行计划。
- `风控官`：只负责质疑，检查仓位、追高、止损、失效线和不交易条件。
- `复盘员`：在复盘时只看原始预测和实际走势，判断命中与错误来源。

## 版本迭代

当用户要精进系统时，按小版本更新，不要一次大改：

- `v1`：结构化分析模板。
- `v2`：加入复盘表和命中统计。
- `v3`：加入关键位有效性评分。
- `v4`：加入仓位与风险收益比模块。
- `v5`：接入真实行情/交易日志，自动生成周报。
- `v6`：把高胜率分析模式沉淀成半自动策略假设。

每次只改一个模块，并记录修改原因、预期改善点、样本外验证结果。
