# 策略跟踪引擎

> 📌 基于 agent-stock-pro 选股建议的模拟交易系统
>
> 初始资金 100 万元，根据每日选股报告的买卖建议模拟执行交易，长期跟踪策略有效性。

---

## 1. 数据文件

所有模拟数据保存在 `dist/strategy/` 目录下：

```
dist/strategy/
├── portfolio.json        # 当前组合状态（核心文件，每日更新）
├── trades/               # 成交记录
│   ├── YYYYMMDD_0001.json  # 每笔成交单独记录

│   └── YYYYMMDD.json
└── reports/              # 策略绩效分析报告

```

## 2. 每日执行流程

每个交易日收盘后（或选股报告生成时），按以下步骤执行：

### 2.0 读取状态

读取 `dist/strategy/portfolio.json`，如不存在则按初始状态创建（100万现金，空仓）。

### 2.1 卖出执行

**检查计划卖出的持仓**，根据今日日期判断：

| 条件 | 操作 | 卖出价格 |
|------|------|---------|
| 今日 = plan_sell_date | **执行卖出** | 取今日收盘价（用 `stock quote` 获取） |
| 今日收盘价 ≤ stop_loss | **止损卖出** | 今日收盘价 |
| 今日收盘价 ≥ take_profit | **止盈卖出** | 今日收盘价 |
| 选股报告建议"清仓/减仓" | **按建议卖出** | 今日收盘价 |
| 无触发条件 | 持有不动 | — |

卖出记录写入 `dist/strategy/trades/`：

```json
{
  "trade_id": "20260610_0002",
  "type": "sell",          // sell / stop_loss / take_profit
  "symbol": "301222",
  "name": "正帆科技",
  "shares": 1000,
  "price": 18.80,
  "amount": 18800,
  "fee": 18.80,            // 佣金+印花税，按 0.1% 估算
  "date": "2026-06-10",
  "pnl": 300,              // 盈亏 = (price - cost_price) * shares - fee
  "pnl_pct": 1.62,
  "reason": "计划日期卖出"
}
```

**交易费用估算**：
- 佣金：双边 0.025%（万2.5），最低 5 元
- 印花税：卖出 0.05%
- 合计约 0.1%（简化计算）

### 2.2 买入执行

**根据当日选股报告的推荐列表执行买入**：

| 条件 | 操作 |
|------|------|
| 推荐股票评分 ≥ 65 且当前未持仓 | 执行买入 |
| 推荐股票已在持仓中 | 不重复买入 |
| 推荐股票评分 < 65 | 不买入 |

买入数量计算：

```
买入金额 = 可用现金 × 推荐仓位%
买入数量 = floor(买入金额 / 挂单2价格 / 100) × 100   // 按整百股取整
```

买入记录写入 `dist/strategy/trades/`：

```json
{
  "trade_id": "20260608_0001",
  "type": "buy",
  "symbol": "301222",
  "name": "正帆科技",
  "shares": 1000,
  "price": 18.50,
  "amount": 18500,
  "fee": 18.50,
  "date": "2026-06-08",
  "score": 72,
  "plan_hold_days": 2,
  "plan_sell_date": "2026-06-10",
  "stop_loss": 17.95,
  "take_profit": 20.35,
  "reason": "主线科技股，资金连续流入"
}
```

### 2.3 更新持仓与现金

```
新现金 = 旧现金 - 买入金额 - 买入费用 + 卖出金额 - 卖出费用
更新持仓列表：移除已卖出的，新增刚买入的
计算总市值 = Σ(持仓.shares × 持仓.current_price)
总权益 = 现金 + 总市值
```

### 2.4 更新 portfolio.json

将最新状态写回 `dist/strategy/portfolio.json`。

### 2.5 保存净值快照

将当日净值快照保存到 `dist/strategy/equity/YYYYMMDD.json`。

## 3. 策略绩效统计

每次更新组合时计算以下指标：

### 3.1 核心指标

```
累计收益率 = (当前总权益 - 初始资金) / 初始资金 × 100%
年化收益率 = ((当前总权益 / 初始资金) ^ (250 / 交易日数) - 1) × 100%
最大回撤 = 历史上净值峰值到谷值的最大跌幅
夏普比率 = (年化收益率 - 无风险利率) / 年化波动率
  无风险利率按 2.5% 计
```

### 3.2 交易统计

```
总交易次数 = 买入次数 + 卖出次数
盈利交易数 / 亏损交易数
胜率 = 盈利交易数 / 总卖出次数 × 100%
平均盈利 = 盈利交易的平均收益率
平均亏损 = 亏损交易的平均收益率
盈亏比 = 平均盈利 / |平均亏损|
平均持仓天数
```

### 3.3 风险指标

```
最大单票亏损
最大连续亏损次数
最大仓位占比
日均换手率
```

### 4.2 Python 辅助脚本

在选股流程中，用 Python 脚本读取 `portfolio.json`、获取行情、执行模拟交易、计算绩效、输出 Markdown 片段。

**关键实现要点**：

1. 使用 `stock quote <代码>` 获取实时价格
2. 解析当日选股报告中的推荐列表
3. 按规则执行买卖决策
4. 计算绩效指标
5. 输出 Markdown 格式的策略跟踪板块

### 4.3 月度汇总

每月最后一个交易日，额外生成一份 `dist/strategy/reports/YYYYMM_report.md`，包含：

- 月度收益曲线图数据
- 交易明细汇总
- 与沪深300等基准的对比
- 策略改进建议

## 5. 约束与纪律

1. **严格按照选股报告建议执行**，不人为干预买卖决策
2. **评分 < 65 的标的不买入**，保持纪律
3. **止损必须执行**，不扛单
4. **不追加买入**（不摊平成本、不金字塔加仓）
5. **同板块持仓不超过总仓位 30%**
6. **单笔买入金额不超过可用现金的 30%**
7. **非交易日不执行任何操作**
8. **所有操作必须留痕**，可追溯、可审计

---

## 6. 实现方式

每日选股流程（`screen.md` §4 生成 PDF 之前）中嵌入策略跟踪步骤：

1. 读取 `portfolio.json`（不存在则初始化）
2. 用 Python 脚本执行买卖决策和绩效计算
3. 更新 `portfolio.json` 和交易记录
4. 在 Markdown 报告末尾追加策略跟踪板块
5. 生成 PDF 时自然包含该板块

Python 脚本模板放在 `dist/strategy/update_strategy.py`，选股流程中调用。
